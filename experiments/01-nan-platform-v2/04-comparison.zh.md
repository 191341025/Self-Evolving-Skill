# v1 与 v2 对比分析

> 基于同一数据库 (nan_platform) 和同一任务集的两次实验数据驱动对比，
> 隔离 Five-Gate v2（置信度衰减模型）相对于 Five-Gate v1（二元日期检查）的效果差异。

---

## 1. 实验设置

| 维度 | v1 | v2 |
|------|----|----|
| 数据库 | nan_platform | nan_platform（相同） |
| 任务集 | 25 个任务（5 轮 x 5） | 25 个任务（5 轮 x 5），同一套 |
| 门控协议 | Five-Gate v1（二元日期） | Five-Gate v2（置信度衰减） |
| Gate 4 模型 | 仅日期新鲜度检查 | C(t) = e^(-lambda x t)，6 种知识类型 |
| 起始状态 | 干净（空 references/） | 从 v1 继承 3 条 tool_experience |
| 语言 | 仅中文 | 双语（英文为默认） |

---

## 2. 知识增长

| 轮次 | v1 行数 | v2 行数 | v1 写入内容 | v2 写入内容 |
|------|---------|---------|------------|------------|
| R0 | ~20 | 144 | 0 | 0（继承 3 条） |
| R1 | ~90 | 173 | Schema + 规则 | Schema + 规则 |
| R2 | ~105 | 176 | 2 条工具笔记 | 1 条工具笔记 |
| R3 | ~110 | 176 | 少量补充 | 1 次纠错 |
| R4 | ~115 | 176 | 0 | 0 |
| R5 | ~120 | 176 | 0 | 0 |

> **说明**：v1 行数为近似值（从演化日志中重建）；v2 在每轮快照中有精确测量。

**观察**：

- v2 起始基线更高（144 行），因为继承了知识并且每条条目的元数据更丰富。
- 两个版本在 R3-R4 趋于收敛，增量接近零。
- v2 在 R0-R1 阶段增长更陡峭（继承知识加速启动），从 R2 开始更平坦。

---

## 3. 门控决策对比

| 指标 | v1 | v2 |
|------|----|----|
| 接受率 | ~30% | ~24% |
| Gate 1 拒绝数 | ~12 | ~15 |
| Gate 2 纠错数 | 0 | 1（room_type） |
| 自纠错次数 | 0 | 1 |
| 衰减元数据 | 无（仅日期） | 完整（类型、lambda、C0） |

**观察**：

- v2 更严格：接受率 24% 对比 30%。置信度衰减模型正确拒绝了更多短暂数据（data_range、data_snapshot 类型，lambda 较高）。
- v2 的 Gate 2 在实验内部捕获了一个枚举错误（R3 纠正了 R1 的 room_type 错误）。v1 在此处从未触发 Gate 2 纠错。
- v2 中每条被接受的条目都携带衰减元数据（知识类型、lambda、初始置信度 C0），支持未来会话中的程序化新鲜度评估。

---

## 4. 质量对比

| 指标 | v1 | v2 |
|------|----|----|
| R5 一次成功率 | 3/4（一个需要结构查找） | 4/4 |
| R5 结构查找次数 | 1 | 0 |
| 实验期间错误数 | 3 | 4（更激进的首次尝试） |
| 错误恢复方式 | 手动检查列 | 相同 + 利用工具经验 |
| 知识准确性 | 良好（未检测到错误） | 更好（Gate 2 捕获并纠正了 1 个错误） |

**观察**：

- v2 在重复轮（R5）实现了完美的 4/4 一次成功率，对比 v1 的 3/4。
- v2 总错误数更多（4 对 3），但这些错误发生在较早轮次，此时 agent 尝试了更激进的首次查询。继承的 tool_experience 使得恢复更快。
- v2 的 Gate 2 ALIGNMENT 检查更有效，捕获了 v1 静默携带的 room_type 枚举错误。

---

## 5. 关键差异

### 5.1 衰减模型的影响

v2 为每条条目标记了类型特定的衰减率。这使得未来的对话能够自动判断知识新鲜度。v1 仅有日期标记，需要人工判断。

| 响应级别 | 置信度范围 | 操作 |
|----------|-----------|------|
| TRUST（信任） | C >= 0.8 | 直接使用 |
| VERIFY（验证） | 0.5 <= C < 0.8 | 使用但验证关键字段 |
| REVALIDATE（重验） | C < 0.5 | 从源重新推导 |

### 5.2 更严格的拒绝

v2 接受率更低（24% 对 30%），正确拒绝了更多短暂数据。六种知识类型的不同 lambda 值提供了细粒度过滤：

- `schema`（lambda=0.003，半衰期 ~231 天）-- 表关系、JOIN 路径
- `tool_experience`（lambda=0.005，半衰期 ~139 天）-- Shell 陷阱、参数技巧
- `business_rule`（lambda=0.008，半衰期 ~87 天）-- 状态约定、枚举
- `query_pattern`（lambda=0.015，半衰期 ~46 天）-- SQL 模板
- `data_range`（lambda=0.035，半衰期 ~20 天）-- MIN/MAX 时间戳
- `data_snapshot`（lambda=0.050，半衰期 ~14 天）-- 行数、分布

### 5.3 自纠错能力

v2 的 Gate 2 在同一实验内捕获了 room_type 枚举错误（R3 纠正了 R1 的错误）。v1 从未针对此问题触发 Gate 2 纠错。这表明 v2 协议对 ALIGNMENT 检查的强调具有实际价值。

### 5.4 更好的 R5 表现

v2 在重复轮实现了完美的 4/4 一次成功率，对比 v1 的 3/4。零次结构查找对比 1 次。继承的 tool_experience 笔记消除了一整类探索性查询。

### 5.5 知识继承

v2 以来自 v1 的 3 条 tool_experience 笔记作为起点，展示了跨实验知识迁移：

1. SQL 工具中 `!=` 运算符必须替换为 `<>`
2. 基于时间的查询前先检查数据时间范围
3. SQL 参数中避免使用中文字符串

这些在实验开始时以 TRUST 置信度（C >= 0.8）通过验证。

### 5.6 元数据丰富度

v2 的最终状态在总量相似的情况下拥有更丰富的元数据。每条条目都可在未来会话中进行过期评估。每条条目的元数据示例：

```html
<!-- decay: type=schema confirmed=2026-03-12 C0=1.0 -->
```

使用公式 `C(t) = 1.0 × e^(-0.003 × t)`，该条目在 231 天后仍有 C=0.50 的置信度。

v1 条目没有此类元数据 -- 新鲜度判断仅为二元的"有日期/无日期"。

---

## 6. 结论

1. **置信度衰减模型有可测量的价值**。该模型为每条条目增加了约 3 行元数据开销，但提供了：
   - 程序化新鲜度评估（无需人工判断）
   - 三级响应协议（TRUST / VERIFY / REVALIDATE）
   - 高衰减类型偏好有效防止数据快照膨胀

2. **Gate 2 ALIGNMENT 在 v2 中更有效**。它捕获了一个 v1 会静默携带的枚举错误。更严格的协议促使 agent 将新知识与现有条目进行比对验证。

3. **跨实验知识迁移成功**。从 v1 继承的 3 条 tool_experience 笔记被成功验证和复用。这是 Skill 知识可跨实验边界存活的首个经验证据。

4. **核心知识结构稳定**。分层设计（schema_map + business_rules + query_patterns + investigation_flows）在两个版本中产生了一致的结果，表明该设计模式在不同门控协议版本下都是稳健的。

5. **收益递减规律一致**。两个版本都展示了相同的收敛模式：R1-R2 快速增长，到 R4-R5 接近零增长。置信度衰减模型不改变基本学习曲线 -- 它提升的是所存储内容的质量。
