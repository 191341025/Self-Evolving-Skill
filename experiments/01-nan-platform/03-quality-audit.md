# 五道门质量审计报告

> 每完成一个 Round 后进行质量审计，评估五道门治理协议的实际效果。

---

## 审计维度

| 维度 | 检查内容 | 健康标准 |
|------|---------|---------|
| **拒绝率** | 五道门拒绝了多少次写入？ | 拒绝 > 接纳（大部分交互不应产生知识变更） |
| **准确性** | 写入的知识是否正确？ | 0 条错误知识 |
| **冗余度** | 有无重复或可合并的条目？ | 无重复 |
| **膨胀度** | references/ 总行数增长是否合理？ | 单文件 < 80行, 总文件 < 8个 |
| **时效标注** | 时间敏感数据是否标注了日期？ | 所有数据快照都带日期 |
| **放置准确** | 知识是否放在了正确的文件中？ | 结构→schema_map, 规则→business_rules |

## 全局总结

### 进化曲线

| Round | 增量(行) | 拒绝率 | Gate 2修正 | 主要产出 |
|-------|---------|--------|-----------|---------|
| R1 | +75 | 67% | 0 | 表结构 + 字典枚举 |
| R2 | +46 | 62% | 1 | SQL模板 + 工具经验 |
| R3 | +12 | 50% | 1 | 业务规则修正 |
| R4 | +21 | 62% | 0 | 调查流程 + Pitfall |
| R5 | +1 | 80% | 0 | 仅1条新JOIN路径 |

### 累计统计
- **总行数**：131 → 289（+158行有效知识）
- **五道门总触发**：33次（12次接纳 / 21次拒绝）
- **总拒绝率**：63.6%（21/33）
- **Gate 2修正**：2次（status规则、设备关联描述）
- **准确率**：100%（0条错误知识存活）
- **膨胀控制**：最大文件83行（< 80行阈值需关注），总文件4个（< 8个上限）

### 设计模式验证结论

1. **五道门有效过滤垃圾知识**：63.6%拒绝率，主要拦截一次性查询结果和时间敏感数据
2. **Gate 2自我纠错能力验证**：2次成功修正已有错误知识（最有价值的Gate）
3. **增量收敛符合预期**：+75→+46→+12→+21→+1，呈明显的成长期→成熟期曲线
4. **重复任务效率显著提升**：Round 5的重复任务全部零探索、直接复用
5. **知识分层合理**：schema_map(结构) → business_rules(规则) → query_patterns(模板) → investigation_flows(流程)逐层激活
6. **Pitfall/Tool Experience是高价值副产品**：非预期的知识类型，实际最实用
