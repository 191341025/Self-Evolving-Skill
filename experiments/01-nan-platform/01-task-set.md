# 实验任务集

> 设计原则：覆盖从简单查询到复杂多步调查的完整难度梯度，每个任务都可能触发五道门决策。
> 任务按轮次执行，每轮结束后检查 references/ 的变化。

## Round 1：结构探索（让 Skill 认识数据库）

| # | 任务 | 预期触发的知识层 |
|---|------|----------------|
| 1.1 | 这个数据库有哪些表？分别是什么业务含义？ | schema_map |
| 1.2 | 楼栋、楼层、房间之间是什么关系？ | schema_map |
| 1.3 | 员工和入驻记录是怎么关联的？ | schema_map |
| 1.4 | 进出记录是通过什么字段关联到员工的？ | schema_map |
| 1.5 | 字典系统是怎么设计的？列出所有字典类型和值 | schema_map + business_rules |

## Round 5：重复与进化验证（重做之前的任务，看 Skill 是否更快更准）

| # | 任务 | 验证目标 |
|---|------|---------|
| 5.1 | 再问一次"各楼栋入住率如何" | 是否直接使用 query_patterns 中已有的模板 |
| 5.2 | 再查一次"员工年龄分布" | 是否跳过了结构探索阶段 |
| 5.3 | 再做一次"幽灵员工"调查 | 是否使用了 investigation_flows 中的流程 |
| 5.4 | 问一个之前没问过的新问题 | 对比 Round 1 时的响应速度和准确度 |

## 每轮检查清单

每轮任务执行完毕后，记录以下内容到进化日志：

1. **references/ 变化 diff**：哪些文件被修改了？新增了什么内容？
2. **五道门决策记录**：每次触发了五道门时，记录决策过程（尤其是被拒绝的）
3. **质量评估**：新增的知识是否准确？是否有膨胀风险？
4. **效率对比**：重复任务时，是否比第一次更快？
