# Experiment #01 — nan_platform 智能楼宇管理数据库

## 实验信息

| 项目 | 值 |
|------|------|
| 目标数据库 | nan_platform (MySQL 8.0) |
| 业务领域 | 智能楼宇 / 工人宿舍管理平台 |
| 数据规模 | 29 表，~95K 行，590MB |
| 使用 Skill | db-investigator |
| 执行日期 | 2026-03-08 |
| 轮次 | 5 轮（结构探索 → 数据查询 → 规则发现 → 复杂调查 → 重复验证） |

## 实验目的

验证 Self-Evolving Skill 设计模式在真实数据库调查场景中的知识治理效果——Skill 是否真的"越用越准确、越用越高效，但不越用越臃肿"。

## 核心结果

| 指标 | 结果 |
|------|------|
| 知识增长 | 131 行 → 289 行（+158 行有效知识） |
| 五道门拒绝率 | 63.6%（21 次拒绝 / 33 次候选） |
| Gate 2 自我纠错 | 2 次（status 字段规则修正、设备关联描述修正） |
| 增量收敛曲线 | +75 → +46 → +12 → +21 → +1 |
| 重复任务效率 | 探索步骤从多步降为 0，直接复用模板 |
| 准确率 | 100%（0 条错误知识存活） |

## 关键发现

1. **五道门有效过滤垃圾知识** — 63.6% 拒绝率，主要拦截一次性查询结果和时间敏感数据
2. **Gate 2 自我纠错能力** — 发现并修正了"所有表 status='1'"的错误规则，这是五道门最有价值的门
3. **增量收敛符合设计预期** — 从 +75 到 +1，清晰的成长期→成熟期曲线
4. **重复任务效率显著提升** — Round 5 重复任务全部零探索、直接复用已有模板和流程
5. **Pitfall 是高价值副产品** — 工具使用经验（`!=`→`<>`、中文 shell 问题）非预期但最实用
