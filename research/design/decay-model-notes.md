> Self-Evolving Skill 设计笔记

## 第一章 衰减函数是什么

### 核心公式

```
C(t) = e^(-λt)
```

- e 是一个固定的数，约等于 2.718，不需要管它从哪来。
- 指数上是负号，意味着 t（时间）越大，整个值越小——这就是「衰减」的来源。

### λ 是衰减的油门

- λ 大 → 油门踩得深 → 衰减快
- λ 小 → 油门轻踩 → 衰减慢

用设计里的数字对比（半衰期 = 置信度降到约 50% 所需天数）：

| 知识类型 | λ 值 | 半衰期 | 第14天剩余 | 第30天剩余 |
|---------|------|--------|-----------|-----------|
| schema（表结构） | 0.003 | 231 天 | 96% | 91% |
| business_rule（业务规则） | 0.008 | 87 天 | 89% | 79% |
| query_pattern（SQL模板） | 0.015 | 46 天 | 81% | 64% |
| data_snapshot（数量统计） | 0.050 | 14 天 | 50% | 22% |

说明：data_snapshot 第14天剩余恰好是 50%，这正是「半衰期」的定义。计算步骤：第一步 0.050×14=0.7，第二步 e^(-0.7)=1/e^0.7≈1/2.013≈0.497≈50%。第30天的 22% 是另一个时间点的独立计算，两者不要混淆。

### 为什么 λ 越大衰减越快？

因为公式里 λ 和 t 是相乘的关系。λ 越大，乘积越大，负号让它变成更大的负数，e 的更大负次方结果就越小。

记住这一个逻辑链就够了：λ 大 → 乘积大 → 负数更负 → 结果更小。

### 补充：为什么用 (β+1)/(α+1) 而不是错误率

#### 直接用错误率 β/(α+β) 会怎样

如果直接用错误率乘以 λ_base：

```
λ_effective = λ_base × β/(α+β)
```

当知识从没出过错时，β=0，错误率=0，代入计算：
1. β/(α+β) = 0/(α+0) = 0
2. λ_effective = λ_base × 0 = 0
3. C(t) = e^(-0×t) = e^0 = 1

结果是置信度永远是 1，完全不衰减。一条知识只要没犯过错，哪怕写入了十年也是 100% 置信度——这不合理。

#### 用加了平滑的错误率 (β+1)/(α+β+2) 会怎样

这个公式结果被锁死在 0~1 之间，λ_effective 只能等于或小于 λ_base，永远无法比原速更快。

用1000次全错的极端情况验算：
1. (β+1)/(α+β+2) = 1001/1002 ≈ 0.999
2. λ_effective = λ_base × 0.999 ≈ λ_base

犯了1000次错，衰减速率几乎还是原速——完全没有惩罚效果。

#### (β+1)/(α+1) 的解决方案

这个公式的结果范围是 0 到无穷大，可以突破 1。错误越多，λ_effective 可以远大于 λ_base，衰减明显加速。

| 场景 | 公式计算过程 | λ_effective | 效果 |
|------|------------|-------------|------|
| 全新知识 α=0, β=0 | (0+1)/(0+1) = 1/1 = 1.0 | = λ_base | 按原速衰减 |
| 10次全对 α=10, β=0 | (0+1)/(10+1) = 1/11 ≈ 0.09 | ≈ λ_base × 0.09 | 衰减极慢 |
| 10次全错 α=0, β=10 | (10+1)/(0+1) = 11/1 = 11.0 | = λ_base × 11 | 衰减极快 |
| 1000次全对 α=1000 | (0+1)/(1000+1) ≈ 0.001 | ≈ λ_base × 0.001 | 几乎不衰减 |
| 1000次全错 β=1000 | (1000+1)/(0+1) = 1001 | = λ_base × 1001 | 衰减极其快 |

#### 一句话总结

(β+1)/(α+β+2) 是错误率，范围 0~1，只能减慢衰减，无法加速。(β+1)/(α+1) 是带保底的错误倾向，范围 0~∞，既能减慢也能加速，这才是设计需要的能力。

## 第二章 贝叶斯是什么

用一句话说：贝叶斯是用「新证据」来更新「已有判断」的方法。

### 现实中的问题

衰减函数只管「时间流逝」，但忽略了「使用反馈」。

一条从来没被验证过的知识，和一条被用了二十次都对的知识，在原公式里老化速度完全一样——这不合理。

### 贝叶斯的操作

只需要记住这三行：
- 每次这条知识被用了，且结果是对的 → α + 1
- 每次这条知识被用了，但结果是错的 → β + 1
- 使用历史置信度 = α / (α + β)

α 是对的次数，β 是错的次数，α+β 是总次数。

用你自己的话说就是：这条知识用了这么多次，有多少次是靠谱的。

| 场景 | α | β | 使用历史置信度 |
|------|---|---|-------------|
| 刚写入，还没用过 | 1 | 1 | 1/2 = 0.50 中立 |
| 用了5次，全部正确 | 6 | 1 | 6/7 ≈ 0.86 较高信任 |
| 用了5次，3对2错 | 4 | 3 | 4/7 ≈ 0.57 中等信任 |
| 用了5次，全部错误 | 1 | 6 | 1/7 ≈ 0.14 基本不信 |

### 为什么初始值是 α=1, β=1 而不是 0, 0？

如果从 0, 0 开始，第一次用之前分母是零，没法算。

设成 1, 1 相当于说：「在没有任何使用记录之前，我对这条知识既不特别信任也不特别怀疑，各给一分。」

这个初始假设在数学上叫先验，中文可以理解为：没有证据之前的默认态度。

## 第三章 两者结合的深层含义

衰减函数解决时间维度：知识会老化。
贝叶斯解决使用反馈维度：知识会被验证或否定。
两件事同时存在，所以应该叠加，不是替换。

### 升级后的核心思路

加入贝叶斯之后，λ 本身变成了一个会被使用记录影响的量，而不是固定值。

有效衰减速率公式：

```
λ_effective = λ_base × (β + 1) / (α + 1)
```

- α 是成功次数，α+1 在分母——成功越多，有效 λ 越小，衰减越慢
- β 是失败次数，β+1 在分子——失败越多，有效 λ 越大，衰减越快

完整公式：

```
C(t) = e^(-λ_effective × t)
     = e^(-λ_base × (β+1)/(α+1) × t)
```

### 三条知识的对比

| 知识 | 使用记录 | λ_effective | 含义 |
|------|---------|-------------|------|
| 知识 A | 从未被调用 | = λ_base | 按原速衰减 |
| 知识 B | 15次全对 (α=15) | ≈ λ_base / 16 | 衰减极慢，三个月后依然很高 |
| 知识 C | 3对2错 (α=3, β=2) | = λ_base × 3/4 | 衰减稍慢，但远不如知识B |

### 一句话总结深层含义

知识通过被使用和验证，可以减缓自己的老化速度。没有被验证过的知识，就按最保守的速度衰减。

这才是两者结合之后真正发生的事——不只是两个数字相乘，而是使用历史改变了衰减的斜率本身。

### 贝叶斯与衰减函数的完整公式链

**第一步：原始衰减**

只考虑时间，λ_base 是固定值，所有知识老化速度一样：

```
C(t) = e^(-λ_base × t)
```

**第二步：引入贝叶斯，让 λ 变活**

```
λ_effective = λ_base × (β+1)/(α+1)
```

α = 成功次数，β = 失败次数，初始都为 0。

| 情况 | 计算结果 | 对衰减的影响 |
|------|---------|------------|
| 全新知识 α=0, β=0 | (0+1)/(0+1) = 1.0 | 按原速衰减 |
| 成功次数多 | 分母大，结果 < 1 | 衰减减慢 |
| 失败次数多 | 分子大，结果 > 1 | 衰减加速 |

**第三步：合并成最终公式**

```
C(t) = e^(-λ_effective × t)
     = e^(-λ_base × (β+1)/(α+1) × t)
```

### 三条核心性质

1. C(t) 永远大于 0，永远不会变成负数。
2. 知识通过持续被验证，可以减缓自己的老化速度。
3. 知识犯错越多，老化速度越快，甚至远超原速。

### 下一个未解决的问题

贝叶斯的加入依赖一个前提：必须有办法拿到知识对错的有效反馈。α 和 β 从哪里来、怎么记录，是下一步需要解决的设计问题。

## 第四章 负指数不会产生负数

### 一个常见的误解

很多人第一次看到 e^(-λt) 时会以为：

「指数是负数，乘出来结果也是负数，而且越来越负。」

这个理解是错的。负指数和负数是两件完全不同的事。

### 负指数的真正含义：取倒数

负次方 = 把结果翻到分母，变成「1 除以它」。

推导过程（以 2.718 的 -3 次方为例）：
1. 2.718^(-3) = 1 / 2.718^3 （负号 = 取倒数）
2. 2.718^3 = 2.718 × 2.718 × 2.718 = 20.078
3. 1 / 20.078 ≈ 0.0498 （结果是正数）

| 指数 | 含义 | 结果 |
|------|------|------|
| e^3 | e × e × e | ≈ 20.09（大于1） |
| e^0 | 任何数的0次方都是1 | = 1 |
| e^(-1) | 1 / e | ≈ 0.368（小于1，但正数） |
| e^(-3) | 1 / e^3 = 1 / 20.09 | ≈ 0.050（更小，仍正数） |
| e^(-∞) | 1 / 无穷大 | → 0（趋近0，永远到不了） |

负指数不管多大，结果永远是正数。

e^(-1000000) = 1 / e^1000000，分母是天文数字，但 1 除以任何正数，结果还是正数。只是无限接近 0，但永远到不了 0。

### 这对设计的意义

置信度 C(t) 永远不会变成负数，也永远不会变成零。

这在数学上天然支持了设计里的一条边界约定：

> 不自动删除任何知识条目，即使 C(t) → 0

置信度衰减到极低，不代表这条知识「消失了」，只是「几乎不可信了」。它还留在那里，等待人工确认是否删除。数学和设计意图在这里完全一致。
